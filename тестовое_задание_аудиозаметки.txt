Приложение для аудиозаметок.

1. Аудиозаметка - аудиофайл с дополнительным заголовком, содержащим
	- название заметки
	- цвет
	- флаг зашифровано ли содержимое
	- хэш пароля
	Расширение файла .audionote

2. Слева панель "репозиториев" - папки где хранятся заметки. Сортировка - в порядке добавления. В нижней части панели должна быть закреплена кнопка "добавить/+". 
	2.1 Если нет репозиториев - на весь экран (включая панель заметок) отображается заглушка и кнопка по центру "добавить репозиторий"
	2.2 При добавлении папки, в ней рекурсивно ищутся файлы .audionote. Новые заметки сохраняются в выбранном репозитории. 
	2.3 Текущий репозиторий подсвечивается в списке.
	2.4 Список репозиториев сохраняется между запусками.

3. Справа список заметок. Каждый элемент списка представляет собой отдельную заметку. Сортировка - от нового (вверху) к старым. В элементе списка есть:
	3.1 Ряд сверху: круглая иконка с цветом заметки, в колонку название заметки, длительность (например 01:25) и позиция воспроизведения. Позицию отображать только при активном воспроизведении (например 00.15 / 01:25). Если заметка зашифрована - отображать на круглой иконке соответствующий символ.
	3.2 Ряд снизу: кнопка старт/стоп, диаграмма звука в виде набора прямоугольников, кнопка удалить
	3.3 Фон всего элемента также заливается цветом с прозрачностью
	3.4 Если заметка зашифрована, то при нажатии на Старт - появляется третий ряд с полем ввода пароля, в которое автоматически ставится курсор. При неправильно введенном пароле отображается сообщение об ошибке (любым удобным способом). При верном пароле - воспроизведение начинается.
	3.5 При старте воспроизведения заметки ее элемент увеличивается в размерах. Позиция воспроизведения отображается закрашиванием диаграммы звука слева. Можно перемещать позицию воспроизведения мышью. При перетягивании позиции воспроизведения мышью звук должен ставиться на паузу и возобновиться при отпускании кнопки мыши. При старте воспроизведения, если одна из записок уже воспроизводится - остановить воспроизведение.
	3.6 При остановке воспроизведения элемент возвращается в изначальное состояние
	3.7 При нажатии на "удалить" - появляется диалог подтверждения. При подтверждении - заметка удаляется из списка, также удаляется файл заметки

Кнопка "Добавить" закреплена в нижней части панели. Если элементов нет - отображать заглушку и кнопку "Добавить" по центру

4. При нажатии на "Добавить" открывается диалог, где можно
	4.1 Ввести название заметки
	4.2 Выбрать цвет иконки
	4.3 Включить шифрование, при включенном шифровании необходимо задать пароль
	4.4 Выбрать устройство записи
	4.4 Кнопка начала/завершения записи. При старте записи отображается бегущая диаграмма звука в виде столбцов и время записи.
	4.5 После завершения записи появляется кнопка сохранить. При нажатии "сохранить" создается файл записи в текущей папке. Заметка добавляется в список 
Панели заметок и репозиториев должны поддерживать изменение размеров (splitview)
 	4.6 При отсутствии устройств записи кнопка записи недоступна.

В существующей реализации есть много различных ошибок: утечки памяти, неправильно выполненные элементы ui, архитектурные проблемы, ошибки безопасности данных, плохой стиль кода.

Задача: доделать недостающую функциональность, исправить существующие ошибки в работе, провести код-ревью и исправить проблемные места.

	2. Добавить аанимации переходов, добавления и удаления элементов
